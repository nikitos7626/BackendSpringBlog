Бэкенд-сервис для платформы личного блога с системой аутентификации и управления контентом.
В проекте реализована кастомная логика работы с базой данных через `EntityManager` и защита API с помощью JWT-токенов.
## Стек технологий

*   **Java 17+**
*   **Spring Boot 3** (Web, Security)
*   **JPA / Hibernate** (Работа с данными через `EntityManager`)
*   **PostgreSQL / MySQL**
*   **JWT (JSON Web Token)** (jjwt-api) — для stateless аутентификации.

## Архитектура и Особенности

Проект следует классической трехслойной архитектуре (Repository -> Service -> Controller), но с особенностями реализации:

### 1. Безопасность (JWT)
Реализован сервис `JwtService`, который отвечает за:
*   Генерацию токенов на основе `UserDetails`.
*   Валидацию и извлечение claims (данных пользователя) из токена.
*   Подпись токенов с использованием алгоритма `HS256`.

### 2. Работа с данными (Repository Layer)
Вместо стандартного `JpaRepository` используется ручное управление персистентностью через `EntityManager`:
*   `blogRepository` использует `em.persist()`, `em.merge()`, `em.remove()` и JPQL-запросы (`createQuery`).
*   Это позволяет гибко управлять транзакциями (`@Transactional`) и жизненным циклом сущностей.

### 3. Бизнес-логика (Service Layer)
*   `serviceBlog`: Управление статьями (CRUD, поиск по автору).
*   `UserService`: Управление пользователями и интеграция с Spring Security.

## Функционал

*   **Auth:** Регистрация, Вход, Получение токена.
*   **Blog:**
    *   Создание и редактирование статей.
    *   Просмотр ленты постов.
    *   Фильтрация статей по конкретному пользователю.
    *   Удаление контента.
